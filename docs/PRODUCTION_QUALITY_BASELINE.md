# Production Quality Baseline

本文件定义 Milestone 3 小范围真实用户 Beta 的解析质量与运行成本验收口径。只记录聚合指标和安全诊断，不保存原始口播、完整分享链接、邮箱、Token 或 Secret。

## 单次生成记录

登录用户完成生成后，`generation_tasks.diagnostics` 可记录：

- 菜谱解析 Provider 与模型。
- ASR Provider 与模型。
- AI / ASR 是否使用 fallback。
- 端到端处理耗时。
- 结构化菜谱质量分。
- DeepSeek 总 Token 数（Provider 返回时）。

历史任务没有诊断数据时保留空对象，不回填推测值。

## Beta 验收样本

第一轮至少收集：

- 10 条普通菜谱正文。
- 5 条无需登录即可访问的小红书公开做饭视频。
- 至少覆盖炒、蒸、炖、凉拌和烘焙中的 3 类。

同一来源重复提交命中缓存时不计为新样本，也不重复计费。

## 第一轮门槛

- 动态菜谱完成率不低于 80%。
- 已完成菜谱质量分不低于 70。
- 正文输入 P95 处理耗时不高于 20 秒。
- 小红书视频 P95 处理耗时不高于 60 秒。
- 真实 Provider fallback 率不高于 20%。
- completed task 必须关联 recipe，生成 recipe 必须同时有 ingredients 和 recipe_steps。
- 不允许出现 Secret 泄露、RLS 绕过或跨用户数据读取。

这些是首轮 Beta 的启动门槛，不是长期 SLA。样本不足时只报告实际数量，不外推成功率。

## 失败分类

失败只记录安全分类：来源不可读、媒体提取失败、ASR 失败、AI 解析失败、质量校验失败、菜谱保存失败、任务中断。不得把 Provider 原始敏感响应直接写入数据库或普通日志。

## Checkpoint A1 完成条件

1. 新生成 task 能在 RLS 下写入安全 diagnostics。
2. 至少一条正文和一条小红书真实生成记录可被聚合读取。
3. 诊断不包含原始口播、完整链接参数或 Secret。
4. 测试、lint、build 和生产路由检查通过。

## 首次生产验证

- 正文样本：质量分 86，无 fallback，约 5.0 秒。
- 小红书公开视频样本：质量分 98，火山 Seed ASR 与 DeepSeek 均无 fallback，约 30.4 秒。
- 当前样本数不足以计算稳定成功率或 P95；Checkpoint A2 继续收集，禁止用这两条样本外推。

## Checkpoint A2 首轮统计

- 小红书公开样本：5 条，3 条完成，2 条失败，成功率 60%。
- 成功样本平均质量分：94.7。
- 成功样本 P95 处理耗时：约 30.9 秒。
- fallback：0。
- 正文样本：2 / 10，当前不足以计算正文成功率和 P95。

当前质量、耗时和 fallback 达标，但小红书完成率未达到 80%，因此不能进入 Beta 发布阶段。

## Checkpoint A2 正文样本统计

- 正文样本：10 条，10 条完成，完成率 100%。
- 平均质量分：92.6。
- P95 处理耗时：约 9.8 秒。
- fallback：0。
- 本轮新增 8 条样本每条只调用一次 DeepSeek，平均质量分 93.5，共 15979 Tokens。
- 样本覆盖炒、煎、煮、炖、凉拌、微波炉、口语化转录和多做法合集。极端无关插话在付费测试前已移除，本轮不把它计作真实 Provider 的噪声清理验收；噪声文本仍有本地样本测试覆盖。

Reviewer 发现首轮输出虽全部通过结构评分，但多个步骤出现原文未提供的估算时长，个别 tips 补充了常识建议。处理规则调整为：

- 博主给出的食材和调料用量必须优先原样保留。
- 原文缺少用量时，可按默认 2 人份估算，但必须标注 `AI估算（按2人份）`。
- 步骤时间和火候没有原文依据时标记为“未说明”。
- tips 只保留作者明确强调的注意事项，不补充常识。

来源事实校验已通过本地自动测试；为了遵守单样本单次付费调用约束，没有重复请求本批 8 条真实样本。

Checkpoint Review：Architect PASS（复用现有 DeepSeek Provider，仅新增一个来源校验边界）；QA PASS（70 tests、lint、build）；Reviewer PASS（用量可执行与来源透明同时满足）；Debug PASS（修复估算 warning 求值顺序）；Release PASS（版本 `0.2.0-working.11`，回滚点 `87362ef`）。

## Checkpoint A3 小红书完成率复验

- 新增公开样本：7 条，7 条完成，完成率 100%。
- 合并 A2 历史样本：12 条，10 条完成，完成率 83.3%，达到 Beta 门槛。
- 6 条本地完整链路平均质量分：86.3；P95 处理耗时：约 48.9 秒；平均耗时：约 44.8 秒。
- 1 条 Vercel Production 完整链路：HTTP 200，约 29.2 秒，质量分 78，DeepSeek 与火山 ASR 均未 fallback。
- 7 条均完成媒体提取、FFmpeg 音轨处理、火山 ASR、DeepSeek 解析和质量校验；没有媒体、ASR、AI 或质量失败。
- Production 样本保留来源明确的 `2 小时`、`10 分钟` 和 `大火`，其余缺失时间与火候标为“未说明”；缺失用量按 2 人份估算并明确标记。
- 第 8 条备用链接没有调用，避免在验收门槛已经满足后继续产生无必要费用。

根因证据：A2 两条失败分别停在媒体获取前和 ASR 后解析阶段；更新到固定校验的 yt-dlp `2026.06.09` 并应用来源事实校验后，新队列未复现。平台页面具有波动性，因此只能确认当前版本与样本组合已消除可复现失败，不能把历史失败完全归因于单一因素。

Checkpoint Review：Architect PASS（沿用单一媒体与解析管线）；QA PASS（真实队列 7 / 7，自动测试、lint、build）；Reviewer PASS（来源准确性与可执行估算兼顾）；Debug PASS（历史失败未在新版队列复现）；Release PASS（版本 `0.2.0-working.12`，代码回滚点 `c5d0c2d`）。
