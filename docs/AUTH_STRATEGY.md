# Auth Strategy

## 1. 当前 MVP 登录策略

- 游客模式优先。
- 不登录也能完整使用首页生成菜谱、loading、菜谱详情、本地收藏、收藏页、风味地图、Station 和我的页本地数据。
- 登录只是为后续云端同步做准备，不作为当前 MVP 的使用门槛。
- 登录后的第一个云端能力是收藏同步。
- 登录后的第二个云端能力是生成记录同步。
- 未登录用户仍然完整可用，收藏继续保存在当前浏览器。
- 未登录用户仍然可以完整使用生成流程，生成记录继续保存在当前浏览器。

## 2. 当前登录方式

- 使用 Supabase Email Magic Link。
- 不需要密码。
- 主要用于开发验证和早期测试，确认 Supabase Auth、Redirect URL、登录状态展示和云端收藏同步可以跑通。

## 3. 为什么当前不做手机号登录

- 手机验证码需要接入短信服务。
- 短信发送会产生额外成本。
- 短信送达率受运营商、地区和风控影响。
- 国内手机号支持需要额外验证和配置。
- 手机号登录涉及风控与合规要求。
- 当前 MVP 阶段应优先验证核心菜谱流程，不适合先投入手机号登录。

## 4. 为什么当前不做微信登录

- 微信登录需要微信开放平台或公众号配置。
- 可能需要资质审核。
- 接入复杂度比 Email Magic Link 更高。
- 更适合产品后期在真实用户规模扩大后再做。

## 5. 后续推荐顺序

- 阶段 1：游客模式 + Email Magic Link 测试
- 阶段 2：云端收藏同步
- 阶段 3：云端生成记录同步
- 阶段 4：真实用户数据和多设备同步完善
- 阶段 5：中国用户登录优化，考虑手机号 / 微信登录
