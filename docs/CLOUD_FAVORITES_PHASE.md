# Cloud Favorites Phase

## 1. 当前收藏策略

- 未登录：收藏继续保存在 localStorage。
- 已登录：优先读写 Supabase `favorites` 表。
- Supabase 读取或写入失败：回退到 localStorage，保证页面不崩溃、游客体验不中断。
- 页面层仍然只使用 recipe slug，不直接暴露 `recipe_id`。

## 2. 为什么不强制登录

- 当前 MVP 的核心是验证菜谱生成、浏览、详情和收藏体验。
- 强制登录会增加新用户试用门槛。
- 游客模式可以让用户先完整体验产品，再决定是否登录同步。
- 登录是增强能力，不是使用门槛。

## 3. 为什么先做云端收藏，不做生成任务云端保存

- 收藏是最轻量、最容易验证的云端同步能力。
- 收藏数据结构简单，只需要用户和菜谱的关联关系。
- 生成任务涉及任务状态、错误重试、真实解析来源和后续 AI 流程，复杂度更高。
- 先做好云端收藏，可以验证 Supabase Auth、RLS 和基础用户数据写入。

## 4. 当前仍未接 AI

- 当前生成流程仍然是 MVP mock 流程。
- 本阶段不接 AI API，不做真实内容解析。
- 这样可以避免成本、稳定性和解析质量风险提前进入 MVP。

## 5. 当前仍未做手机号 / 微信登录

- 当前登录方式仍然是 Supabase Email Magic Link。
- 没有接入手机号验证码。
- 没有接入微信登录。
- 中国用户登录方式会放到后续阶段评估。

## 6. 后续阶段

- 云端生成任务。
- AI 解析。
- 多设备同步完善。
- 中国用户登录方式优化，包括手机号 / 微信登录。
